# /pr - プルリクエスト作成コマンド

SDDワークフロー用のPRを作成します。

## 使用方法

```
/pr [type]
```

- `type`: `spec`（仕様レビュー用）または `impl`（実装レビュー用）
  - 省略時: ブランチ名から自動判定

## 実行内容

1. 現在のブランチと変更内容を確認
2. 未コミット変更があればコミット
3. リモートへプッシュ
4. PR内容を生成（タイトル、サマリー、テストプラン）
5. ユーザー確認後、PRを作成
6. PR URLを表示

## PRテンプレート

### 仕様策定用（spec）

```markdown
## Summary
- {アクションID} の仕様を策定

## レビュー観点
- [ ] ユーザーストーリーが明確か
- [ ] ACがEARS記法で記述されているか
- [ ] スコープが適切か
```

### 実装用（impl）

```markdown
## Summary
- {アクションID} を実装

## Test plan
- [ ] 全ACのテストが通過
- [ ] TDDサイクルを遵守

## AC確認
- [x] AC1
- [x] AC2
```

## 例

```
/pr
→ ブランチ名から自動判定してPR作成

/pr spec
→ 仕様レビュー用PRを作成

/pr impl
→ 実装レビュー用PRを作成
```

## エラー処理

- 変更がない場合: 作業継続または既存コミットでPR作成を提案
- 既にPRがある場合: 既存PR更新または新規作成を提案
- gh CLI未認証: `gh auth login` を案内
