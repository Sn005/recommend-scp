# EPIC-001: 技術検証（PoC）

## 概要

SCP推薦システムの技術的実現可能性を検証するためのPoC（Proof of Concept）。

## ユーザーストーリー

**ペルソナ**: 開発者（本プロダクトの開発チーム）

**目的**: SCP推薦システムの技術的実現可能性を検証する

**価値**: 本格実装前にリスクを特定し、技術選定の妥当性を確認できる

**理由**:
- SCP Data APIのデータ構造・品質が未知
- Embedding + 明示的タグのハイブリッド推薦が「感覚的に正しい」結果を返すか検証が必要
- Supabase pgvectorのパフォーマンスが実用レベルか確認したい

## スコープ

### 含む

- SCP Data API からのデータ取得
- EN記事のみ（10〜50件程度）
- Embedding生成（OpenAI text-embedding-3-small）
- 明示的タグ抽出（LLM使用）
- pgvectorによるベクトル検索
- 類似度検索の結果確認
- ε-greedy の概念設計のみ

### 含まない

- 日本支部スクレイピング
- 全記事の取得
- ローカルモデル検証
- 人手によるタグ補正
- 本番環境構築
- UI実装
- ε-greedyの実装

## 技術選定

| 項目 | 選定 | 費用 |
|------|------|------|
| データソース | SCP Data API | 無料 |
| 対象言語 | EN のみ | - |
| Embedding | OpenAI text-embedding-3-small | 〜数円（PoC） |
| タグ抽出LLM | Claude or GPT | 別途 |
| ベクトルDB | Supabase pgvector | 無料枠内 |
| 実装言語 | TypeScript | - |
